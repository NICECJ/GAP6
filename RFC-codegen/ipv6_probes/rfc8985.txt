Mechanism Name: TLP Loss Probe

Condition: When the PTO timer expires and there is no other previous loss probe still in flight, and the sender has obtained an RTT measurement since the last loss probe transmission or the start of the connection

Receive Filter: ACK or SACK for the loss probe segment

Success Condition: The target responds with an ACK or SACK that acknowledges the loss probe segment

Failure Condition: No ACK or SACK is received within the RTO period

Prerequisites: ["PTO timer expiration", "No other loss probe in flight", "RTT measurement since last loss probe or connection start"]

Confidence: High

RFC Sentence: When the PTO timer expires, the sender MUST check whether both of the following conditions are met before sending a loss probe: 1. First, there is no other previous loss probe still in flight. 2. Second, the sender has obtained an RTT measurement since the last loss probe transmission or the start of the connection, whichever was later.

Mechanism Name: TLP Recovery Detection

Packet Flow: Sender -> Receiver: TLP retransmission segment, Receiver -> Sender: ACK with DSACK or DupAck

Transport Layer Protocol: TCP

Application Layer Protocol: None

Receive Filter: ACK with DSACK or DupAck

Success Condition: ACK with DSACK or DupAck received, indicating both original and TLP retransmission segments arrived

Failure Condition: No ACK with DSACK or DupAck received, indicating at least one segment was lost

Prerequisites: ["TLP retransmission has been sent"]

Confidence: High

RFC Sentence: If the TLP retransmission is spurious, a receiver that uses DSACK would return an ACK that covers TLP.end_seq with a DSACK option (Case 1).  If the receiver does not support DSACK, it would return a DupAck without any SACK option (Case 2).
